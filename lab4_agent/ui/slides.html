Khái niệm

LangGraph Router là node phân loại ý định người dùng, dùng add_conditional_edges() để điều hướng luồng đến đúng specialist node - mỗi node có tools & model riêng.

# Router function đọc state, trả về tên node đích def route_by_intent(state) → str: return { "trip_planner": "trip_node", "tour_guide" : "tour_node", "off_topic" : "rejection_node", }.get(state["intent"], "rejection_node") builder.add_conditional_edges("classifier", route_by_intent)
Vai trò
  • Cổng vào thông minh - đúng người, đúng việc, đúng model
  • Cô lập tool - mỗi route chỉ thấy tools của mình
  • Tối ưu chi phí và model hợp tác vụ - Thay đổi model, thông số... tùy theo tác vụ
  • Bảo mật cứng - off-topic từ chối bằng code, không qua LLM
  • Dễ mở rộng - thêm route = thêm 1 node + 1 edge

Slide 4 · Lỗi thường gặp

Những Pitfalls
cần tránh

Lỗi & Fix
⚠Classifier trả nhãn ngoài whitelist
LLM trả "trip" thay vì "trip_planner" → router không khớp → từ chối nhầm.
if intent not in VALID: intent = "off_topic"
⚠Model echo lại <user_query> tag
Input quá ngắn → model lặp lại tag thay vì xử lý nội dung.
re.sub(r"<user_query>.*?</user_query>", "", content)
⚠Vòng lặp tool vô tận
Tool raise exception → LLM không nhận thông báo lỗi → gọi lại mãi.
Tools phải return chuỗi lỗi, không raise exception
⚠State thiếu key khi invoke()
AgentState thêm field intent nhưng không truyền khi gọi graph.
graph.invoke({"messages":[...], "intent":""})
⚠add_conditional_edges thiếu mapping key
Router trả giá trị không có trong dict mapping → LangGraph runtime error.
.get(intent, "rejection_node") # luôn có fallback
